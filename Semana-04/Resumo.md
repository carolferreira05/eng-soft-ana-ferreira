# Engenharia de Software - Semana 4
## Capítulo 03: Desenvolvimento Ágil de Software (Resumo)

**Nome:** Ana Carolina Moreira Ferreira  
**Curso:** Tecnologia em Sistemas para Internet  
**Referência:** Engenharia de Software (9ª edição) - Ian Sommerville

---

Na contextualização do início do capítulo, Sommerville diz que o rápido desenvolvimento dos softwares se dá em função de novas oportunidades e competitividade empresarial, e que é muito difícil ter um conjunto de requisitos estáveis devido ao ambiente de altas mudanças.

**Características das abordagens para o desenvolvimento rápido de software:**

- Ausência de especificação detalhada do sistema, documentação minimizada, são definidos apenas os requisitos mais importantes para o sistema.
- Todos os stakeholders são envolvidos em cada versão para que sugestões possam ser levadas em consideração.
- Foco em software funcional em vez de documentação extensa, colaboração com stakeholders e adaptação contínua a mudanças.

**Métodos ágeis:** métodos de desenvolvimento incremental, onde o software é desenvolvido como uma série de incrementos, cada uma incluindo uma nova funcionalidade. Em geral, são pequenos incrementos que são apresentados a cada curto período de tempo aos clientes, envolvendo-os no desenvolvimento, minimizando a documentação (comunicação informal).

Abordagens como planejamento cuidadoso são de extrema importância em projetos de grande porte, porém em casos de projetos de médio e pequeno porte, tal abordagem na maioria das vezes acaba prejudicando o desenvolvimento, uma vez que gasta-se mais tempo em análises do que no desenvolvimento. Com tal insatisfação, foram propostos métodos ágeis, que permitiam maior foco no desenvolvimento do software.

**Métodos ágeis:** Extreme Programming, Scrum, Crystal, Desenvolvimento de software Adaptativo e Desenvolvimento Dirigido a Características.

**Princípios dos métodos ágeis:** envolvimento do cliente, entrega incremental, valorização das pessoas e interações mais do que processos e ferramentas, aceitar mudanças, manter a simplicidade.

Após o desenvolvimento, o envolvimento do cliente no projeto normalmente é perdido durante as manutenções, uma vez que as mudanças não são contínuas.

Enquanto as abordagens ágeis de desenvolvimento incorporam outras atividades no projeto e na implementação, as abordagens dirigidas a planos identificam estágios diferentes do processo de software com saídas associadas, ocorrendo interações com documentos para estabelecer comunicação entre os estágios.

---

No método ágil **Extreme Programming** práticas como o desenvolvimento iterativo a níveis extremos são aplicadas, os requisitos são expressos como cenários, programadores trabalham em pares e desenvolvem testes para cada tarefa.

**Série de práticas/princípios do Extreme Programming** que refletem os princípios dos métodos ágeis: planejamento incremental, pequenos releases, projeto simples, desenvolvimento test-first, refatoração, programação em pares, propriedade coletiva, integração contínua, ritmo sustentável e cliente no local.
